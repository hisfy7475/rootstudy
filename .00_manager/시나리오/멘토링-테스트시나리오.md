## 1. 관리자 시나리오 — 멘토 관리

### TC-A01. 멘토 등록

**경로**: `/admin/mentoring/mentors`

| # | 행동 | 기대 결과 |
|---|------|----------|
| 1 | 관리자 로그인 → 사이드바 "멘토링 관리" 클릭 | `/admin/mentoring` 페이지 이동, 주간 캘린더 표시 |
| 2 | 상단 또는 사이드바에서 "멘토 관리" 이동 | `/admin/mentoring/mentors` — 멘토 목록 (초기 빈 목록) |
| 3 | "멘토 추가" 버튼 클릭 | 등록 폼 표시 |
| 4 | 아래 정보 입력 후 "저장" 클릭 | 저장 성공, 목록에 새 멘토 표시 |

**입력값 (멘토 1)**:

| 필드 | 값 |
|------|-----|
| 이름 | `김수학` |
| 주요 과목 | `수학` |
| 소개 | `수능 수학 전문 멘토입니다.` |
| 프로필 이미지 URL | (비워둠) |
| 활성 | `체크` |

| # | 행동 | 기대 결과 |

## 3. 학생 시나리오

### TC-S02. 멘토링 신청 (정상 플로우)

| # | 행동 | 기대 결과 |
|---|------|----------|
| 1 | 남은 좌석이 있는 슬롯 카드 클릭 | `/student/mentoring/{slotId}/apply` 신청 폼 페이지 이동 |
| 2 | 슬롯 상세 정보 확인 | 유형, 멘토명, 과목, 날짜·시간, 장소, 남은 좌석 표시 |
| 3 | 메모에 `교재 질문 있습니다` 입력 후 "신청하기" 클릭 | `applyMentoring` 호출 → 성공 |
| 4 | 페이지 이동 | `/student/mentoring/my` (내 신청 내역)으로 리다이렉트 |
| 5 | `mentoring_applications` 확인 | `status: 'pending'`, `user_id`: 학생 본인, `student_id`: 학생 본인, `note` 저장 |

| # | 행동 | 기대 결과 |

### TC-S04. 중복 신청 방지

| # | 행동 | 기대 결과 |
|---|------|----------|
| 1 | 이미 `pending` 상태 신청이 있는 동일 슬롯에 다시 신청 | `이미 신청한 슬롯입니다.` 에러 |
| 2 | 이미 `confirmed` 상태 신청이 있는 동일 슬롯에 다시 신청 | `이미 신청한 슬롯입니다.` 에러 |

| 상태 변화 | booked_count 변화 |
|----------|------------------|

| 거절 (pending → rejected) | -1 |
| 삭제 (application 삭제 시) | -1 |
