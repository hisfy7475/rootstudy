## 1. 목적 / 범위

- 학생에게 단어 꾸러미 기반의 **영단어 예습** 및 **일일 영단어 시험**을 제공한다.
- 관리자가 개발사 요청 없이 **단어 꾸러미·단어·문제를 직접 등록·관리**할 수 있게 한다.
- 레거시의 고정 `LEVEL 1~3`을 폐기하고 확장 가능한 **단어 꾸러미(pack)** 구조로 전환한다.

### 범위 외 (이번 단계 제외 제안)

- 단어 자동 채점 외의 학습 게임/스펠링 입력형 문제(현 스펙은 4지선다 객관식 한정)
- 레거시 응시내역 데이터 이관(별도 협의)

## 3. 학생 사용자 흐름

```
영단어 테스트 홈
 ├─ 예습하기
 │   └─ 단어 꾸러미 선택 → 예습 단어 목록 확인
 │
 ├─ 시험보기
 │   └─ 시험 안내 확인 → 꾸러미 선택 → 시험 시작
 │       └─ 40문제 / 10분 → (수동/자동) 제출 → 결과 확인
 │
 └─ 응시내역
     └─ 과거 결과 및 오답 확인
```

### 3-1. 단어 꾸러미 선택 (예습/시험 공통)
- 명칭은 "난이도 선택" → **"단어 꾸러미 선택"**.
- 관리자가 **공개(public)** 상태로 둔 꾸러미만 노출.
- 준비중(preparing) 꾸러미 노출 정책 (기본안 채택): **비활성 버튼 + "준비중" 배지로 노출**.
- 한 번에 **하나만** 선택. 선택 시 테두리/배경/체크로 강조.
- 미선택 시 "시작하기" 비활성, 선택 시 활성.
- 시험 시작 전까지 뒤로가기로 재선택 가능.

### 3-2. 시험 안내 화면 (문구는 레거시 문구 활용)
> "영단어 시험은 10분간 진행됩니다. 시험에 응시할 단어 꾸러미를 선택해 주세요."

안내사항:
- 하루 1회(계정 기준), 40문제, 10분 제한, 시간 종료 시 자동 제출
- 제출 후 재응시 불가, 미선택 문항은 오답
- **매주 금요일에는 월~목 틀린 단어가 시험에 포함됨**
- 시험 중 앱을 종료해도 제한시간은 계속 진행

- **금요일 분기:** 금요일에도 화면 흐름은 동일하다(꾸러미 선택 → 시작). 다만 진입 시점이 금요일이면 안내 화면에 "이번 주 오답 복습 시험" 배지/문구를 추가로 노출하고, 선택한 꾸러미 기준으로 오답이 포함됨을 안내한다.

### 3-3. 시험 진행 화면 구성
| 영역 | 내용 |
| --- | --- |
| 상단 | 뒤로가기/시험 종료 버튼, 선택 꾸러미명, 남은 시간, `현재/전체` 진행 |
| 본문 | 영어 단어(문제), 한국어 뜻 4지선다(정답 1개) |
| 하단 | 다음 문제 / (마지막) 제출 버튼 |

| 항목 | 값 |
| --- | --- |
| 문제 수 | 40 |
| 제한시간 | 10분 |
| 형태 | 객관식 4지선다 |
| 문구 | 영어 단어 |
| 선지 | 한국어 뜻 |
| 정답 수 | 1 |
| 응시 횟수 | 하루 1회 |

### 3-4. 시험 중 이탈·재접속 (중요 — 서버 권위 타이머)
- 뒤로가기 시 종료 여부 확인.
- 화면 이탈해도 타이머 계속(서버 `started_at` 기준 절대시간).
- 제한시간 내 재접속: **기존 문제·답안 복원, 새 10분 부여 안 함.**
- 제한시간 경과 후 진입: 결과 화면으로 이동, **자동 제출 처리.**
- 네트워크 단절: 기기 임시 저장 답안으로 복원.
- 시험 시작 버튼 연타에도 **중복 시험 미생성** (멱등 보장).

**답안 저장 전략 (재접속 복원의 전제):**
- 학생이 보기를 고를 때마다 해당 문항 `selected`를 서버에 **즉시 upsert**(낙관적 저장)하고, 동시에 **기기 로컬에도 백업**한다.
- 네트워크가 끊기면 로컬 답안으로 진행하다가, 복구 시 서버와 동기화한다.
- 재접속 시에는 서버 저장 답안을 기준으로 화면을 복원한다(로컬과 충돌 시 서버 우선, 단 서버에 비어 있고 로컬에 있으면 로컬 보충).

### 3-5. 시험 결과 / 오답 확인 화면
제출(수동/자동) 직후, 그리고 응시내역에서 과거 시험을 열 때 동일하게 보여준다.

| 영역 | 내용 |
| --- | --- |
| 요약 | 점수(예: `32 / 40`), 정답 수 / 오답 수, 제출 방식(정상/자동), 시험 구분(일반/금요일 오답), 꾸러미명, 응시일 |
| 문항 리스트 | 문항별 영어 단어 · 정답 뜻 · 내가 고른 답 · 정답 여부(O/X). 미선택은 오답으로 표기 |
| 필터 | **오답만 보기** 토글 |

- 합격선·등급·배지는 없음(점수 표시만).
- 결과는 **출제 당시 스냅샷**(`vocab_exam_questions`) 기준으로 렌더링하므로, 이후 단어가 수정·비활성화돼도 과거 결과는 그대로 유지된다.
- 이미 제출된 시험에 다시 진입하면 새 응시가 아니라 **이 결과 화면**으로 보낸다.

## 4. 데이터 모델 제안 (Supabase / Postgres)

> 컨벤션상 `src/types/database.ts`도 수작업 갱신 필요. 시각/날짜는 `getStudyDate` 기준(KST 06:00 경계).

### 4-1. `vocab_packs` — 단어 꾸러미
| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | uuid PK | |
| `name` | text | 사용자 노출명 (예: LEVEL 1, KICE 빈출) |
| `code` | text UNIQUE | 내부 관리 코드 (엑셀 업로드 매칭 키) |
| `description` | text null | 관리자 메모 |
| `status` | text | `preparing` / `public` / `hidden` / `disabled` |
| `display_order` | int | 노출 순서 |
| `publish_start_at` | timestamptz null | 공개 시작 |
| `publish_end_at` | timestamptz null | 공개 종료 |
| `created_at` / `updated_at` | timestamptz | |

- 노출 조건: `status='public'` AND (`publish_start_at` is null OR ≤ now) AND (`publish_end_at` is null OR ≥ now). **상태는 크론으로 바꾸지 않고 조회 시점에 계산**한다(공개 종료일이 지나면 status는 public이어도 자동으로 안 보임).
- **"사용 가능한 단어 수" 정의:** 이 꾸러미에 연결된 단어 중 `is_active=true`인 단어 수. (목록의 "등록된 단어 수"는 활성·비활성 합계)
- 공개 가능 조건: 사용 가능한 단어 ≥ 40 (보기 생성을 위해 권장 41+ → 4-3 참고). 미만이면 공개 전 경고.
- **삭제 정책:** 시험 기록(`vocab_exams`)이 하나라도 있으면 hard delete 금지 → `disabled`(사용중지)로만 전환. 기록이 전혀 없는 꾸러미만 완전 삭제 허용.

### 4-2. `vocab_words` — 단어 (꾸러미와 독립)
| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | uuid PK | |
| `english` | text | 영어 단어 |
| `korean_primary` | text | 대표 뜻(정답으로 출제) |
| `korean_extra` | text null | 추가 뜻 |
| `problem_group` | text null | 문제 그룹 |
| `is_active` | bool | 사용 여부 (false면 신규 출제/예습 제외) |
| `created_at` / `updated_at` | timestamptz | |

- **중복 정책:** `lower(english)` 기준 동일 단어로 판단(부분 unique 인덱스 권장). 한 단어를 여러 꾸러미에 연결.

### 4-3. `vocab_pack_words` — 꾸러미↔단어 (M:N)
| 컬럼 | 타입 |
| --- | --- |
| `pack_id` | uuid FK → vocab_packs |
| `word_id` | uuid FK → vocab_words |
| PK | (`pack_id`, `word_id`) |

- **보기(distractor) 생성:** 한 문항의 보기 4개 = **정답 뜻 1개(항상 포함)** + 오답 보기 3개. 오답 보기는 **정답 단어와 `problem_group`(품사 그룹)이 하나라도 겹치는(교집합) 다른 단어**의 한국어 뜻에서, **정답 뜻과 같은 문자열은 제외**하고 **서로도 중복되지 않게** 랜덤 3개를 뽑는다. (학생이 품사만으로 정답을 추려내지 못하게 함. 정답 뜻 자체는 절대 제외되지 않으므로 "정답이 보기에 없는" 경우는 생기지 않는다.)
- **부족 시 처리:** 같은 그룹 내 다른 고유 뜻이 3개에 못 미치면(소규모 그룹·`problem_group` 미입력 등), 부족분만 **같은 꾸러미 전체 단어의 고유 뜻 풀**에서 보충한다. 단일 누적 집합으로 정답 제외·뜻 중복 제거·정확히 3개 확보를 함께 보장한다. 그래도 꾸러미 전체 고유 뜻이 4개(정답+3) 미만이면 그 꾸러미는 **출제 불가**로 보고 시험 시작 가드(6장 규칙 5)에서 차단한다.
- **알려진 한계:** ① `problem_group`이 비어 있는(NULL) 단어는 교집합이 없어 그룹 제한 없이 전체 풀에서 보기를 채운다(의도된 fallback). ② 같은 품사로 보기를 좁히면 정답의 유의어가 보기에 낄 확률이 올라가고, 채점은 정확 문자열 일치이므로 학생이 "또 다른 옳은 뜻"을 골라도 오답 처리될 수 있다(레거시 동작 그대로이며 정확매칭 채점은 기존과 동일 — 뜻 표기 품질로 관리).
- 따라서 정답 단어 외에 **서로 다른 고유 뜻이 최소 3개** 더 필요 → 공개 가능 조건을 "활성 단어 ≥40 **그리고** 고유 한국어 뜻 ≥4"로 본다. 실무상 활성 단어 **41개 이상**을 권장하고, 40 경계에서 뜻 중복이 많으면 공개·시작 단계에서 경고/차단한다.

### 4-4. `vocab_exams` — 시험 응시(헤더)
| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | uuid PK | |
| `student_id` | uuid FK | |
| `pack_id` | uuid FK | |
| `exam_type` | text | `normal` / `friday_review`(금요일 오답) |
| `exam_date` | date | `getStudyDate(started_at)` 결과 — 하루 1회 제약 키 |
| `started_at` | timestamptz | 타이머 기준 |
| `submitted_at` | timestamptz null | |
| `submit_type` | text | `in_progress` / `normal` / `auto` |
| `score` | int null | 정답 수 |
| `total` | int | 40 |

- **`exam_date` 계산 = `getStudyDate(started_at)`.** 학습일 경계는 KST 06:00이며, `getStudyDate`/`getStudyDayBounds`의 기존 경계 규칙(06:00 이상은 당일, 03:00~06:00 리셋 구간은 직전 학습일로 귀속)을 그대로 따른다. **영단어 기능이 별도 경계를 정의하지 않는다** — 직접 Date 계산 금지(CLAUDE.md 도메인 시간 규칙).
- **하루 1회 제약:** `UNIQUE(student_id, exam_date)` — 계정 기준 하루 1회, 꾸러미 무관. 금요일도 별도 시험이 아니라 그날의 일일 시험 1개에 오답이 포함되는 형태이므로 동일 제약으로 충분.
- **멱등 시작(범위 명시):** 시작 시 `student_id=나 AND exam_date=오늘(=getStudyDate(now)) AND submitted_at is null`인 레코드가 있으면 신규 생성 대신 **그것을 재개**한다. 동시 요청(버튼 연타)은 `INSERT ... ON CONFLICT(student_id, exam_date) DO NOTHING` + 재조회로 처리하여 UNIQUE 제약이 race를 흡수하게 한다(애플리케이션 select-then-insert에 의존하지 않음).
- **과거 미완료 분리:** 멱등 재사용 조건에 `exam_date=오늘`을 명시하므로, **어제 이전의 미완료(`in_progress`) 레코드는 오늘 시작에 절대 재사용되지 않는다.** 과거 미완료분은 아래 lazy/크론 마감으로 정리한다.
- **미완료 시험 마감(lazy finalize + 크론 백업):** 학생이 시간 초과 후 다시 안 들어오면 레코드가 `in_progress`로 남는다. **이 시험을 읽는 시점**(학생 재진입 / 관리자 응시내역·결과 조회)에 `now > started_at + 10분 && submitted_at is null`이면 그 자리에서 자동제출(`submit_type='auto'`)로 확정 채점한다. **단독으로는 영원히 안 읽힐 수 있으므로, `daily-reset` 크론에서 만료된 미마감분을 일괄 마감하는 것을 기본(권장)으로 둔다.** 관리자 "진행 중" 필터는 (a) 크론이 만료분을 이미 auto로 바꿔 두므로 실제로 진행 중인 것만 보이고, (b) 조회 중 마감되어 목록에서 빠지는 경우는 새로고침 시 반영된다.

## 5. 관리자 기능 명세

### 5-1. 시험 응시내역 관리
- 검색: 시작일/종료일, 센터, 그룹, 학생명, 자리번호, 단어 꾸러미, 시험 구분(일반/금요일오답), 제출상태(진행중/정상/자동), 점수 범위.
- 목록 컬럼: 학생, 그룹, 자리번호, 시험날짜, 시작시간, 제출일시, 단어 꾸러미, 시험구분, 점수(예 32/40), 제출방식, 상세보기.
- **엑셀 다운로드:** 현재 검색조건 **전체**(현재 페이지 한정 X). 학생명/그룹/자리번호/시험일/꾸러미/점수/제출시간 포함.

### 5-2. 단어 꾸러미 관리
- 목록: 꾸러미명, 관리코드, 등록 단어 수, 사용가능 단어 수, 공개상태, 표시순서, 등록/수정일, 수정.
- 등록 필드: 꾸러미명(필수), 관리코드(필수·중복불가), 설명(선택), 표시순서(필수), 상태(필수), 공개 시작/종료일(선택).
- 규칙:
  - 공개 전 사용가능 단어 < 40 → **경고**.
  - 시험기록 존재 꾸러미는 **완전 삭제 금지 → 사용중지(disabled)** 처리.
  - 추가 즉시 앱 업데이트 없이 사용자 화면 반영(웹 동적 조회).

### 5-3. 단어·문제 관리
- 검색: 사용여부, 꾸러미, 영어단어, 한국어뜻, 문제그룹, 등록일.
- 목록: 번호, 꾸러미, 영어단어, 한국어뜻, 문제그룹, 사용여부, 등록/수정일, 수정.
- 레벨 컬럼 → **꾸러미 컬럼**으로 변경.
- **문제 그룹:** `단어 꾸러미`와 별개 컬럼으로, 단어의 **품사 그룹**(명사=1, 형용사=2, 부사=3, 동사=4, 숙어=5)을 나타낸다. `vocab_words.problem_group` 텍스트 필드에 `"1, 2"`처럼 쉼표 멀티값으로 저장(한 단어가 여러 품사에 동시 소속 가능). **출제 시 오답 보기를 같은 품사 그룹(교집합)에서 뽑는 데 사용**한다(4-3 보기 생성 참고). 목록·검색·엑셀에서는 라벨(명사·형용사…)로 노출.
- 중복 정책: 대소문자만 다르면 동일 단어. 동일 단어 다중 꾸러미 연결 권장(새로 중복등록 지양).
- 사용여부: 사용→사용안함 가능. 사용안함은 신규 시험/예습 제외, 기존 결과는 당시 값 유지.

### 5-4. 단어 엑셀 업로드
- 컬럼: 꾸러미코드, 꾸러미명, 영어단어, 한국어 대표뜻, 추가뜻, 문제그룹, 사용여부.
- **단어↔꾸러미 연결 로직:** 한 행 = `(꾸러미코드 + 영어단어)` 한 쌍. 영어단어는 `lower(english)` 기준으로 이미 있으면 **재사용**(새로 만들지 않음)하고, 해당 꾸러미에 **연결만 추가**(`vocab_pack_words`). 같은 단어를 여러 꾸러미에 넣으려면 꾸러미코드만 다른 여러 행으로 올린다.
- 중복(=이미 존재하는 영어단어) 처리 옵션:
  - **기존유지:** 단어 필드는 건드리지 않고 꾸러미 연결만 추가.
  - **기존수정:** 대표뜻·추가뜻·문제그룹·사용여부를 행 값으로 갱신 + 꾸러미 연결 추가. (영어 철자 자체는 키이므로 변경 안 함)
  - **중복행 제외:** 그 행 스킵(연결도 추가 안 함).
- 검증·결과:
  - 업로드 전 형식 검사. 존재하지 않는 꾸러미코드 → 오류 안내.
  - 영어단어/한국어 대표뜻 공란 → 등록 제외.
  - 오류 행 + 사유를 엑셀로 재다운로드 제공.
  - 일부 오류 시 **정상 행만 등록**(클라이언트 선호) 기본, 단 관리자에게 확인.
- 표준 .xlsx 템플릿(헤더·열 순서 고정)은 우리가 제작·배포한다.

## 6. 비즈니스 규칙 정리 (구현 핵심)

1. **타이머는 서버 권위.** 남은시간 = 10분 − (now − `started_at`). 클라이언트 표시는 보조.
2. **하루 1회 = 학습일(KST 06:00 경계) 기준, 계정당 1회(꾸러미 무관).** `getStudyDate` 사용. 03:00~06:00 리셋 구간 주의.
3. **출제 = 선택 꾸러미 활성 단어에서 매번 랜덤 40개.** 진도/순차 개념 없음(별도 진도 테이블 불필요). 같은 단어가 며칠 뒤 재출제될 수 있음 — 차근차근 학습은 예습 기능이 담당.
4. **금요일 오답 시험 (정밀 정의):** 금요일은 별도 시험이 아니라 그날의 일일 시험 1개.
   - **"이번 학습주 월~목" 경계:** `getWeekStart(now)`(월 06:00)부터 목요일 학습일 끝까지. 캘린더 자정이 아니라 **학습일 경계** 기준.
   - **"금요일" 판정:** `getStudyDate(now)`의 요일이 금요일인지로 판단(자정이 아닌 학습일 기준).
   - 출제 풀 = 위 기간에 **선택한 그 꾸러미** 문항 중 `오답 또는 미선택`이었던 `word_id` 집합(중복 제거). 단, **현재 그 꾸러미에 연결되어 있고 `is_active=true`인 단어로 한정**(그 사이 사용안함/꾸러미 이탈한 단어는 제외).
   - 이 오답 단어를 우선 배치하고 모자란 만큼 같은 꾸러미 활성 단어에서 랜덤 보충해 40개. 오답이 41개 이상이면 그중 **40개를 랜덤 선택**.
   - **월~목 오답이 0개면 일반 랜덤 시험과 동일**하되 시험 구분만 `friday_review`로 기록.
   - 문항 생성 시 단어·정답·보기는 **그 시점 값으로 스냅샷**(과거 시험의 스냅샷을 재사용하지 않음).
5. **시험 시작 가드:** 시작 시점에 선택 꾸러미의 **활성 단어 ≥40 그리고 고유 한국어 뜻 ≥4**를 모두 만족하지 못하면 시작을 막고 안내("일시적으로 응시할 수 없습니다"). (4-3 보기 생성 조건과 동일 기준)
6. **보기(distractor) 3개 = 정답 단어와 `problem_group`이 겹치는(교집합) 다른 단어의 한국어 뜻에서 랜덤** 추출(정답 뜻과 동일 문자열 제외·중복 제거, 부족 시 꾸러미 전체에서 보충).
7. **문항 스냅샷 보존**으로 사후 단어 수정과 무관하게 결과 일관성 유지.
8. **멱등 시작**으로 버튼 연타·재접속 시 중복 시험 방지.
9. **미완료 시험은 조회 시점 lazy 마감**(4-4 참고).
10. **공개 노출 = status + 공개기간 + 사용가능단어 40개** 동시 충족.
11. **예습은 횟수 제한 없음**, 선택 꾸러미의 활성 단어 전체를 노출.
12. **단어 꾸러미는 전 센터 공통 콘텐츠**(센터별 분리 없음). 응시내역의 센터 필터는 응시 학생 소속 기준.

### 8-2. 클라이언트 확인사항

- **(필수) 상벌점·학부모 연동 여부:** 영단어 시험 점수를 기존 **상벌점/주간 리포트/학부모 알림**과 연동할 필요가 있는지. 별도 요청이 없으면 **연동 없이 독립 동작**으로 진행. (연동 시 범위·일정 추가 산정 필요)
- **(확인) 관리자 검색의 "그룹" 정의:** 응시내역 검색·목록의 **"그룹(소속 그룹 또는 층)"** 에 해당하는 데이터가 현재 스키마에는 단독 컬럼으로 없다(센터=`branch_id`, 자리번호=`student_profiles.seat_number`는 존재). 기존 **`student_types`(학생 분류)** 를 그룹으로 매핑하는 것을 기본안으로 하되, 클라이언트가 말하는 "그룹/층"이 별개 개념이면 컬럼 신설 필요. → 기본안: `student_types` 재사용.
- **(확인, 기본값 있음) 지점관리자 단어 관리 권한:** 단어·꾸러미 마스터를 **슈퍼관리자 전용**으로 둘지(기본안), 지점관리자에게도 허용할지. 응시내역 조회는 지점관리자=자기 지점 학생분으로 제한(기본안).
