# 07. 제출 직전 최종 체크리스트

> 제출 버튼 누르기 전에 한 줄씩 체크. 거절되면 재심사 5~7일 손해 → 미리 한 번 다 점검.

---

## 공통

- [ ] `app.json` 의 `version`(1.0.0) / `ios.buildNumber` / `android.versionCode` 가 EAS 빌드와 일치
- [ ] `app.json` 의 `bundleIdentifier` / `package` = `com.rootstudy.app` (양쪽 동일)
- [ ] `assets/icon.png` 1024×1024, 알파 없음
- [ ] `assets/splash-icon.png` 정상
- [ ] `assets/android-icon-foreground/background/monochrome.png` 모두 존재
- [ ] HTTPS 외 통신 없음 (NSAppTransportSecurity NSAllowsArbitraryLoads = false)
- [ ] 사용 안 하는 권한 제거 — **`RECORD_AUDIO`는 1.0에서 빼고 1.1에서 추가 권장**
- [ ] 데모 계정 (학생/학부모) 생성 + 출결·채팅·급식 데이터 시드
- [ ] 개인정보처리방침 페이지(`/privacy`)와 계정삭제 페이지(`/account-deletion`) 운영 도메인에서 200 OK
- [ ] `.well-known/apple-app-site-association`, `.well-known/assetlinks.json` 실값(Team ID, SHA256) 반영 (보류 중이면 1.0에서는 노출만 동작 확인 — 거절 사유는 아님)

## App Store (iOS)

### Build

- [ ] EAS production 빌드 → `.ipa` Transporter 또는 EAS Submit으로 업로드
- [ ] App Store Connect 의 빌드 목록에 새 빌드가 "Processing" 후 사용 가능
- [ ] Build 선택 후 "Export Compliance" 응답 (Yes → exempt, HTTPS만)

### Metadata (02 파일 그대로)

- [ ] Name `루트스터디`
- [ ] Subtitle `학습관리·출결·소통을 한 번에`
- [ ] Promotional Text 입력
- [ ] Description 입력
- [ ] Keywords 입력 (100자 이내)
- [ ] Support URL `https://www.rootstudy.co.kr`
- [ ] Privacy Policy URL `https://www.rootstudy.co.kr/privacy`
- [ ] Copyright `© 2026 RootStudy`
- [ ] Category — Primary `Education`
- [ ] Age Rating 모든 항목 None / UGC = Yes (moderated)

### App Privacy

- [ ] "Get Started" 후 04 파일의 표대로 데이터 신고 완료
- [ ] "Tracking" = No (모든 항목)

### Review Information

- [ ] Sign-in required = Yes + 데모 계정 정보
- [ ] Contact (성/이름/전화/이메일)
- [ ] Notes — 02 파일의 본문 그대로 (외부결제·카메라·UGC 사유 포함)

### Screenshots

- [ ] 6.7" iPhone 4~7장 업로드
- [ ] (선택) App Preview 동영상

### Submit

- [ ] "Manually release this version" 또는 "Automatically release after approval" 선택
- [ ] **Submit for Review** 클릭

## Google Play (Android)

### Build

- [ ] EAS production 빌드 → `.aab` 업로드
- [ ] **Internal Testing** 트랙에 먼저 올려서 본인 디바이스로 설치 테스트
- [ ] Closed Testing → Production 단계 승격 권장 (첫 출시는 Production 직배도 가능하지만 권장 안 함)

### Main store listing (03 파일 그대로)

- [ ] App name `루트스터디 - 학습관리·출결·소통`
- [ ] Short description (80자)
- [ ] Full description (4000자)
- [ ] App icon 512×512
- [ ] Feature graphic 1024×500 (필수!)
- [ ] Phone screenshots 4~7장

### App content

- [ ] Privacy policy URL
- [ ] App access (데모 계정 정보 + Instructions)
- [ ] Ads = No
- [ ] Content rating IARC 설문 완료 → 04 파일대로
- [ ] Target audience 13세 이상
- [ ] **Data safety** 04 파일의 표대로 입력 + Save and submit

### Pricing & distribution

- [ ] Free
- [ ] Korea, Republic of 만 활성화

### Release

- [ ] Production 트랙에 빌드 추가
- [ ] Release notes 입력 (06 파일의 v1.0.0)
- [ ] **Save → Review release → Start rollout to Production**

## 흔한 거절 사유 / 사전 대응

| 거절 사유 | 대응 |
|----------|------|
| **Apple 5.1.1 — 데모 계정 빈 데이터** | 데모 계정에 출결·채팅·급식 미리 입력 |
| **Apple 4.1 — UGC 신고/차단 기능 없음** | 채팅 메시지 길게 누르면 "신고" 메뉴 노출 (또는 02 Notes에 1:1·관리자 모니터링 명시) |
| **Apple 3.1.1 — 외부 결제 사용** | "실물 음식 결제" 임을 02 Notes에 명시 |
| **Apple 2.1 — 카메라 권한 사유 부적절** | `app.json` infoPlist 의 한국어 사유 문구 그대로 (이미 OK) |
| **Apple 4.5.4 — Push 권한 즉시 요청** | 앱 시작 직후가 아닌 사용자 액션(예: 첫 채팅 진입) 후 요청하도록 코드 검토 |
| **Google 데이터 안전 자기신고 누락** | 04 파일의 표 그대로 입력 |
| **Google 정책 위반 — 사용 안 하는 권한** | `app.json` 에서 `RECORD_AUDIO` 제거 |
| **Google 13세 미만 정책** | Target audience 13+ 체크, "Designed for Families" 미참여 |
