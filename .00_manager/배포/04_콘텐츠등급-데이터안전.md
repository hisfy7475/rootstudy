> 잘못 답하면 등급이 올라가거나 거절 사유가 됨. **표 그대로** 콘솔 설문에서 클릭.

| 카테고리 | 답변 |

| Cartoon or Fantasy Violence | None |
| Realistic Violence | None |
| Prolonged Graphic or Sadistic Realistic Violence | None |
| Profanity or Crude Humor | None |
| Mature/Suggestive Themes | None |
| Horror/Fear Themes | None |
| Medical/Treatment Information | None |
| Alcohol, Tobacco, or Drug Use or References | None |
| Simulated Gambling | None |
| Sexual Content and Nudity | None |
| Graphic Sexual Content and Nudity | None |
| Contests | None |
| Unrestricted Web Access | **No** *(WebView가 있지만 도메인 화이트리스트로 rootstudy.co.kr 만 로드)* |
| Gambling | No |
| User Generated Content | **Yes** — moderated *(채팅 메시지·이미지 첨부가 있음. 신고/차단 기능 필요. 06 항목 참고)* |

**예상 등급**: `4+`

> ⚠️ "User Generated Content"를 Yes로 두면 신고/차단 기능이 있어야 함. 현재 채팅이 1:1(학생↔담임/지점)이라 공개 게시판이 아니므로 큰 위험은 낮지만, **메시지 신고 또는 사용자 차단 버튼**을 채팅에 추가해두면 안전. 현재 미구현 시 02 파일의 "심사 메모"에 "공개 게시판이 아닌 1:1 사적 채팅이며 관리자 모니터링됨"으로 보강.

## C. Google Play "Data safety" 섹션 — 정답표

Play Console → App content → Data safety → "Manage" → 아래 표대로 체크.

### C-1. Overview (개요 단계)

| 질문 | 답변 |
|------|------|
| Does your app collect or share any of the required user data types? | **Yes** |
| Is all of the user data collected by your app encrypted in transit? | **Yes** (모든 통신 HTTPS / TLS 1.2+) |
| Do you provide a way for users to request that their data be deleted? | **Yes** → URL: `https://www.rootstudy.co.kr/account-deletion` |

### C-2. Data types — 수집·공유 항목

각 항목은 "Collected (수집)" 와 "Shared (제3자 공유)" 를 따로 묻는다. **공유는 모두 No**.

| 카테고리 | 데이터 타입 | Collected | Shared | Optional? | 사용 목적 |
|----------|------------|-----------|--------|-----------|----------|
| Personal info | Name | ✅ | ❌ | Required | Account management, App functionality |
| Personal info | Email address | ✅ | ❌ | Required | Account management |
| Personal info | Phone number | ✅ | ❌ | Required | Account management, Communications |
| Personal info | User IDs | ✅ | ❌ | Required | Account management, App functionality |
| Photos & videos | Photos | ✅ | ❌ | Optional | App functionality (채팅 첨부) |
| Messages | Other in-app messages | ✅ | ❌ | Optional | App functionality (채팅) |
| Files & docs | Files & docs | ✅ | ❌ | Optional | App functionality (채팅 첨부) |
| App activity | App interactions | ✅ | ❌ | Required | Analytics, App functionality (학습 통계) |
| App info & performance | Crash logs | ✅ | ❌ | Required | App functionality |
| App info & performance | Diagnostics | ✅ | ❌ | Required | App functionality |
| Device or other IDs | Device or other IDs | ✅ | ❌ | Required | App functionality (푸시 토큰) |

### C-3. **수집 안 함** — 명시적으로 No

- Location (Approximate / Precise): ❌
- Contacts: ❌
- Calendar: ❌
- Health & fitness: ❌
- Audio recordings: ❌ *(권한은 있지만 현재 사용 안 함; 사용 시 즉시 갱신)*
- Search history, Web browsing history: ❌
- Financial info (Payment info, Purchase history, Credit score): ❌ *(NICEPay PG가 처리, 앱은 결과 토큰만 수신)*
- Race / ethnicity / religion / sexual orientation 등 민감정보: ❌

### C-4. Security practices

| 질문 | 답변 |
|------|------|
| Data is encrypted in transit | **Yes** |
| You provide a way for users to request that their data be deleted | **Yes** → `https://www.rootstudy.co.kr/account-deletion` |
| Independent security review (옵션) | (선택 사항, 지금은 No) |
| Committed to follow Google Play Families Policy | N/A (Designed for Families 미참여) |

> 🔄 향후 RECORD_AUDIO 권한을 실제 사용하게 되면 "Audio recordings = Yes"로 즉시 갱신. 현재 권한만 선언되어 있고 사용 안 함.

## D. Permissions 사용 사유 (앱에 이미 선언된 권한 정리)

| 권한 | 플랫폼 | 사유 (사용자에게 표시되는 문구) |
|------|--------|----------------------|
| `NSPhotoLibraryUsageDescription` | iOS | 채팅에 이미지를 보내기 위해 사진 라이브러리 접근이 필요합니다. |
| `NSCameraUsageDescription` | iOS | 채팅에 사진을 촬영해 보내기 위해 카메라 접근이 필요합니다. |
| `NSUserNotificationsUsageDescription` | iOS | 공지·채팅·결제 안내 알림을 받기 위해 사용됩니다. |
| `RECORD_AUDIO` | Android | (현재 미사용; 추후 음성 첨부 기능 추가 시 사용 예정) |
| `POST_NOTIFICATIONS` | Android 13+ | 공지·채팅·결제 안내 알림 |

> 사용 안 하는 권한은 가급적 `app.json`에서 제거하는 것이 안전. RECORD_AUDIO는 v1.1 이후로 미루고 1.0에서는 빼는 것을 권장. (코드 수정 필요)
