## 2. 프로젝트 특성

- **플랫폼**: 크로스플랫폼 앱 (React Native Expo) + 관리자 웹 (Next.js) 혼합
  - 앱 쉘(1~4) + 급식·멘토링 앱(5,7): **크로스플랫폼 → 노드당 0.8일, 테스트 0.30**
  - 급식 관리·멘토링 관리·기존 개선(6,8,9): **웹 → 노드당 0.5일, 테스트 0.15**
- **레거시 연동**:
  - 기존 Supabase DB 연동 (이미 운영 중) → API 연동 수준 (1.2x)
  - 해당 노드: 급식 신청(5번, 10개) + 멘토링 신청(7번, 9개) + 멘토링 관리(8번, 12개) + 기존 개선(9번, 10개) = 41개
- **외부 연동**:
  - 나이스페이먼츠 결제 + 환불 API: 복잡한 API (+3일, 신규 연동)
  - FCM/APNs 푸시 인프라: 복잡한 API (+2일)
  - 카카오 알림톡 (NCloud SENS, 기존 연동): 간단한 API (+0.5일, 멘토링 신규 템플릿)

## 4. 최종 견적

```
N = 25주 (여유 기간 기준)

버퍼제외 = ceil(25 ÷ 1.25) = ceil(20) = 20주
최소 = ceil(20 ÷ 2) = 10주
권장 = ceil((10 + 25) ÷ 2) = ceil(17.5) = 18주
여유 = 25주
비용 = 25 × 100 = 2,500만원
기획변경제한 = ceil(18 ÷ 2) = 9주차 (권장 기준)
```

- **개발 기간**: 18주 (권장) / 25주 (여유)
- **개발 비용**: 2,500만원 (1주당 100만원, 여유 기간 N 기준)

### 검증

- 노드 82개 → 크로스플랫폼+웹 혼합 기준 예상 20~30주 → **25주 여유 / 18주 권장 적정** ✓
- 나이스페이먼츠 신규 연동 + FCM/APNs 인프라 구성은 복잡도 높아 추가 시간 포함
- 기존 8주 플랜 대비 멘토링 시스템 추가로 약 5주 증가 (기존 8주 플랜: ~20주 여유 기준 1,900만원)
- **결론**: 적정함
