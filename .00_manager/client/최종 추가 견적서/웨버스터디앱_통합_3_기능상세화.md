# Step 3: 기능 상세화 — 웨버스터디앱 통합 플랜

> 기존 8주 플랜 + 멘토링/클리닉 신청을 통합한 화면·작업 단위 상세화 문서입니다.
> 레거시 iOS/Android 앱 코드는 참조만, 실제 구현은 Next.js + React Native(Expo) 신규 개발.

---

## 1. 앱 쉘 시스템 상세

### 1-1. Expo 앱 초기 구성

**화면/작업 구성**:
- Expo 프로젝트 생성 및 기본 설정: package.json, app.json, eas.json 구성
- iOS/Android 빌드 환경 설정: 번들 ID, 패키지명, 서명 키
- 앱 아이콘 설정: iOS(1024×1024) + Android(여러 해상도) 아이콘 적용
- 스플래시 스크린 설정: 로딩 화면 이미지 및 배경색 설정

**주요 처리**:
- Expo EAS Build 파이프라인 구성 (iOS/Android 동시)
- 환경변수 관리 (.env.local, eas.json secrets)
- TestFlight / 내부 테스트(Android) 배포 채널 구성

---

### 1-2. WebView 임베드 및 딥링크

**화면/작업 구성**:
- WebView 메인 화면: 기존 Next.js 웹 URL 임베드, 로딩 인디케이터
- 뒤로가기 처리: 앱 내 뒤로가기 인터셉트 (WebView history vs 앱 종료)
- Safe Area 처리: 상단 노치·하단 홈 인디케이터 여백 대응
- 딥링크 라우팅: 외부 링크 → 내부 URL 매핑 처리

**주요 처리**:
- `react-native-webview` 설정 (쿠키, JS 허용, User-Agent 주입)
- 앱 실행 시 초기 URL 결정 (자동 로그인 여부에 따라 분기)
- Android 하드웨어 뒤로가기 버튼 처리
- 딥링크 scheme 등록 (iOS: `weverstudy://`, Android: Intent Filter)

---

### 1-3. App Store / Google Play 배포

**화면/작업 구성**:
- App Store Connect 앱 등록: 앱 이름, 설명, 스크린샷, 카테고리
- Google Play Console 앱 등록: 동일 정보 입력
- 심사 제출 처리

**주요 처리**:
- iOS 스크린샷 (6.7인치, 6.5인치, 5.5인치 각 3장 이상)
- Android 스크린샷 (폰, 태블릿)
- 개인정보처리방침 URL 등록
- App Store 심사 리젝 대응

---

## 2. 푸시 알림 시스템 상세

### 2-1. 푸시 인프라 설정

**화면/작업 구성**:
- FCM 프로젝트 설정: Firebase 콘솔, google-services.json (Android)
- APNs 인증서 설정: Apple Developer → p8 키 발급 및 등록
- Expo Notifications 모듈 설정: 토큰 등록 요청 화면 (권한 요청 모달)
- 푸시 토큰 저장 API: 디바이스별 토큰을 Supabase에 저장

**주요 처리**:
- 앱 최초 실행 시 알림 권한 요청 (iOS는 명시적 요청 필수)
- 토큰 갱신 감지 및 업데이트 처리
- 포그라운드 / 백그라운드 / 종료 상태별 알림 처리 분기

---

### 2-2. 공지사항 푸시 발송

**화면/작업 구성**:
- 관리자 공지 등록 시 푸시 발송 트리거 (웹 서버 액션)
- 푸시 발송 대상 선택: 전체 / 학생만 / 학부모만 / 특정 지점

**주요 처리**:
- Supabase Edge Function 또는 Next.js API Route로 Expo Push API 호출
- 대상 유형별 디바이스 토큰 조회 후 일괄 발송
- 발송 실패 토큰 정리 처리

---

### 2-3. 채팅 메시지 푸시 발송

**화면/작업 구성**:
- 채팅 메시지 전송 시 상대방 푸시 알림 발송
- 알림 탭 → 해당 채팅방으로 이동 (딥링크)

**주요 처리**:
- 채팅방 참여자 중 앱이 백그라운드/종료 상태인 사용자에게만 발송
- 채팅방 open 상태 체크 (포그라운드 시 푸시 미발송)
- 발신자 이름 + 메시지 미리보기 포함

---

### 2-4. 멘토링 신청 푸시 알림

**화면/작업 구성**:
- 학생/학부모 신청 완료 → 관리자에게 신청 접수 알림
- 관리자 확정/거절 → 학생/학부모에게 결과 알림

**주요 처리**:
- 알림 유형별 메시지 템플릿 구성
- 딥링크: 알림 탭 → 해당 신청 내역 화면으로 이동

---

## 3. 자동 로그인 / 세션 관리 상세

### 3-1. 자동 로그인

**화면/작업 구성**:
- 앱 실행 시 로딩 화면: 토큰 유효성 확인 중 스플래시 유지
- 자동 로그인 성공: 바로 웹 메인 화면으로 이동
- 자동 로그인 실패: 로그인 화면으로 이동

**주요 처리**:
- Expo SecureStore에 Supabase refresh_token 저장
- 앱 실행 시 토큰 조회 → Supabase 세션 복원 → WebView 쿠키 주입
- 토큰 만료 시 자동 갱신 (rotate refresh token)
- 로그아웃 시 SecureStore 토큰 삭제

---

### 3-2. 비밀번호 재설정

**화면/작업 구성**:
- 비밀번호 재설정 요청 화면: 이메일 입력 폼
- 인증 코드 입력 화면: 6자리 코드 입력 (기존 링크 방식 → 코드 방식 변경)
- 새 비밀번호 입력 화면

**주요 처리**:
- Supabase OTP 이메일 발송 (6자리 코드)
- 코드 검증 후 새 비밀번호 설정
- 앱 환경에서 딥링크 리다이렉션 없이 동작하도록 코드 방식 전환

---

## 4. 파일·사진 첨부 기능 상세

### 4-1. 채팅 파일·이미지 첨부

**화면/작업 구성**:
- 채팅 입력 영역 첨부 버튼 탭 → 선택 모달: 카메라 촬영 / 갤러리 선택 / 파일 선택
- 이미지 미리보기 화면: 선택 이미지 확인 후 전송
- 전송 중 진행 표시: 업로드 프로그레스 바
- 채팅 메시지 내 이미지 썸네일 표시
- 이미지 풀스크린 뷰어: 채팅 내 이미지 탭 → 전체화면 확대

**주요 처리**:
- Expo ImagePicker (갤러리/카메라) 권한 요청 처리
- Expo DocumentPicker (파일 선택)
- Supabase Storage에 업로드 후 URL 저장
- 이미지 압축 처리 (최대 1080px, 품질 80%)
- 파일 다운로드 처리 (Expo FileSystem)

---

### 4-2. 공지사항 파일·이미지 첨부 (관리자)

**화면/작업 구성**:
- 공지 작성 화면 내 첨부 영역: 이미지·파일 첨부 버튼
- 첨부 파일 목록 표시: 파일명, 크기, 삭제 버튼
- 공지 상세에서 첨부파일 다운로드 버튼

**주요 처리**:
- 관리자 웹에서 Supabase Storage 업로드
- 파일 타입 제한 (이미지: jpg/png/gif, 파일: pdf/doc/xlsx 등)
- 최대 파일 크기 제한 (10MB)
- 앱에서 공지 상세 조회 시 파일 다운로드/미리보기

---

## 5. 급식 신청 시스템 상세

### 5-1. 급식 목록 조회 (학생·학부모)

**화면/작업 구성**:
- 급식 메인 화면: 날짜 선택 탭 (이번 주/다음 주), 중식/석식 탭 전환
- 급식 상품 카드 목록: 센터명, 식사유형, 날짜, 가격, 신청기간, 잔여여부
- 신청 상태 배지: 신청가능 / 신청완료 / 마감 / 기간외

**주요 처리**:
- 현재 로그인 사용자의 센터 기준 급식 상품 조회
- 신청 가능 기간 실시간 체크 (서버 시간 기준)
- 이미 신청한 상품 상태 표시

---

### 5-2. 급식 신청 및 결제

**화면/작업 구성**:
- 급식 상세 화면: 상품 정보, 메뉴 상세, 신청 버튼
- 신청 확인 화면: 신청자 정보(자동 입력), 상품 정보, 결제 금액 확인
- 나이스페이먼츠 결제 화면: WebView 내 결제 모듈 실행
- 결제 완료 화면: 신청 완료 메시지, 신청 내역 바로가기

**주요 처리**:
- 중복 신청 방지 체크 (서버 측)
- 나이스페이먼츠 결제 요청 파라미터 구성 (주문번호, 금액, 상품명)
- 결제 성공/실패 콜백 처리 및 상태 저장
- 결제 완료 후 신청 DB 저장 + 확인 알림톡 발송

---

### 5-3. 급식 취소·환불

**화면/작업 구성**:
- 신청 내역 목록 화면: 신청 목록, 상태 배지 (신청완료/취소불가/취소완료)
- 신청 상세 화면: 상품 정보, 결제 정보, 취소 버튼
- 취소 확인 모달: 환불 금액 안내, 취소 버튼
- 취소 완료 화면

**주요 처리**:
- 취소 가능 기한 체크 (식사일 2일 전까지, 서버 시간 기준)
- 나이스페이먼츠 환불 API 호출 (전액 환불)
- 환불 완료 후 신청 상태 업데이트 + 취소 확인 알림톡

---

### 5-4. 신청 내역 조회 (학생·학부모)

**화면/작업 구성**:
- 신청 내역 목록 화면: 날짜 역순 정렬, 상태별 필터 탭

**주요 처리**:
- 현재 사용자 기준 전체 신청 내역 조회
- 상태별 필터 (전체/신청완료/취소)

---

## 6. 급식 관리 시스템 상세 (관리자 웹)

### 6-1. 급식 상품 등록·관리

**화면/작업 구성**:
- 급식 상품 목록 화면: 등록된 상품 리스트, 상태(판매중/마감/예정)
- 급식 상품 등록 화면: 센터 선택, 식사유형(중식/석식), 날짜, 가격, 판매기간 입력
- 급식 상품 수정 화면
- 메뉴 입력 화면: 해당 날짜의 메뉴 텍스트 입력 (1~2종)

**주요 처리**:
- 센터별 날짜/유형 중복 상품 방지 검증
- 판매 기간 설정 (시작~종료 날짜+시간)
- 메뉴 저장 및 학생 화면 반영

---

### 6-2. 급식 신청 현황 관리

**화면/작업 구성**:
- 신청 현황 목록 화면: 날짜·상품별 신청자 목록, 인원 집계
- 관리자 강제 취소 화면: 신청자 선택 → 취소·환불 처리

**주요 처리**:
- 날짜/센터/상태별 필터 조회
- 관리자 강제 취소 → 나이스페이먼츠 환불 API 호출
- 취소 확인 알림톡 발송

---

## 7. 멘토링/클리닉 신청 시스템 상세 (학생·학부모)

### 7-1. 멘토링 캘린더 화면

**화면/작업 구성**:
- 멘토링 메인 캘린더 화면: 월 단위 달력 표시, 신청 가능 날짜 점 표시
- 날짜 탭 → 해당 날짜 슬롯 목록으로 이동

**주요 처리**:
- 신청 가능한 슬롯이 있는 날짜 강조 표시
- 오늘 이전 날짜 비활성화 처리
- 잔여 좌석 0인 슬롯 날짜는 만석 표시

---

### 7-2. 슬롯 선택 화면

**화면/작업 구성**:
- 슬롯 목록 화면: 선택한 날짜의 가용 슬롯 리스트
  - 슬롯 카드: 시간대, 멘토(선생님) 이름, 과목, 유형(멘토링/클리닉), 잔여 좌석
- 잔여 좌석 0 슬롯 → 비활성화(그레이) 표시

**주요 처리**:
- 잔여 좌석 실시간 조회 (Supabase 실시간 구독 또는 폴링)
- 이미 신청한 슬롯 시간대 중복 방지 체크

---

### 7-3. 신청 처리

**화면/작업 구성**:
- 신청자 정보 확인 화면: 학생 이름(자동 입력), 신청 사유 입력 (선택)
- 신청 완료 화면: 신청 접수 확인 메시지, 내역 바로가기

**주요 처리**:
- 중복 신청 방지 (동일 슬롯 시간대 기준)
- 신청 데이터 저장 (상태: 신청중)
- 관리자에게 신청 접수 푸시 알림 발송
- 학생/학부모에게 신청 접수 확인 알림톡 발송

---

### 7-4. 신청 내역 조회 및 취소

**화면/작업 구성**:
- 신청 내역 목록 화면: 상태 탭 (전체/신청중/확정/취소), 날짜 역순 정렬
- 신청 상세 화면: 슬롯 정보, 멘토 정보, 신청 상태, 취소 버튼
- 취소 확인 모달: 취소 주의사항 안내

**주요 처리**:
- 취소 가능 시간 전 체크 (관리자 설정 기준)
- 취소 완료 후 슬롯 잔여 좌석 +1 복구
- 관리자에게 취소 알림 발송

---

## 8. 멘토링 스케줄 관리 상세 (관리자 웹)

### 8-1. 멘토 데이터 관리

**화면/작업 구성**:
- 멘토 목록 화면: 등록된 멘토(선생님) 리스트, 상태(활성/비활성)
- 멘토 등록/수정 화면: 이름, 담당 과목, 유형(멘토링/클리닉/양쪽)

**주요 처리**:
- 멘토 등록·수정·비활성화 처리
- 슬롯 등록 시 멘토 드롭다운에서 선택 가능하도록 연동

---

### 8-2. 슬롯 등록·관리

**화면/작업 구성**:
- 슬롯 등록 화면: 멘토 선택, 날짜, 시작·종료 시간, 과목, 유형(멘토링/클리닉), 정원 입력
- 슬롯 목록 화면: 날짜별 슬롯 리스트, 수정·삭제 버튼

**주요 처리**:
- 동일 멘토·동일 시간대 슬롯 중복 방지 검증
- 슬롯 삭제 시 기존 신청자 처리 정책 (신청 존재 시 삭제 불가 또는 일괄 취소)
- 슬롯 일괄 등록 (반복 슬롯: 매주 X요일 등)

---

### 8-3. 주간 캘린더 뷰

**화면/작업 구성**:
- 주간 캘린더 화면: 7일 × 시간축 그리드
  - 슬롯 블록: 멘토명, 과목, 신청 인원/정원 표시
  - 슬롯 클릭 → 신청자 목록 모달

**주요 처리**:
- 주간 단위 슬롯 데이터 조회 (멘토별 필터 옵션)
- 슬롯 블록 색상 코딩 (멘토링/클리닉 구분)
- 잔여 좌석 부족 슬롯 강조 표시

---

### 8-4. 신청 관리 (확정·거절·강제취소)

**화면/작업 구성**:
- 신청 내역 목록 화면: 날짜별·학생별·상태별 필터, 페이지네이션
- 신청 상세 모달: 학생 정보, 슬롯 정보, 신청 사유
- 확정/거절 버튼: 클릭 시 확인 모달 → 처리 + 알림 발송
- 관리자 강제 취소: 확정 건 취소 처리 + 알림

**주요 처리**:
- 신청 확정 → 학생/학부모에게 확정 알림톡 + 푸시
- 신청 거절 → 학생/학부모에게 거절 알림톡 + 푸시
- 강제 취소 → 취소 알림 발송

## 9. 기존 웹 기능 개선 상세

### 9-1. 몰입도 개선

**화면/작업 구성**:
- 지난 시간 추가 입력 화면: 수동 학습시간 입력 UI
- 핸드폰 제출 항목 추가: 몰입도 입력 시 핸드폰 제출 여부 체크박스
- 학생 리포트 노출: 학생 화면에서 자신의 몰입도 리포트 확인 화면
- 과목별 학습시간 띠그래프: 주간 리포트 내 과목별 비율 시각화

**주요 처리**:
- 지난 시간 입력 권한 범위 설정 (당일 or N일 이내)
- 띠그래프 컴포넌트 구현 (과목별 색상 구분)

---

### 9-2. 상벌점 개선

**화면/작업 구성**:
- 30점 도달 알림: 상벌점 합산 → 30점 도달 시 관리자 알림 + 처리 화면
- 분기별 초기화 설정 화면: 초기화 일정 설정 (분기 기준)
- 100점 상품권 처리 화면: 100점 도달 시 상품권 지급 + 초기화
- 학부모 메뉴 상벌점 탭 추가

**주요 처리**:
- 상벌점 임계값 트리거 로직 (30점 강제퇴원, 100점 상품권)
- 분기별 배치 초기화 cron 구성

---

### 9-3. 스케줄 개선

**화면/작업 구성**:
- 일회성 스케줄 자동 삭제: 해당 날짜 경과 후 자동 삭제 처리
- 정기 스케줄 기간 표기: 목록에서 반복 기간 표시
- 관리자 스케줄 화면 레이아웃 개선
- 결석 기능 추가: 학생별 결석 처리 화면

**주요 처리**:
- 일회성 스케줄 만료 처리 배치 (daily cron)
- 결석 상태 추가 및 출석 집계에 반영

---

### 9-4. 채팅 개선

**화면/작업 구성**:
- 출석부 → 채팅방 연결: 출석부 학생 행 클릭 → 해당 채팅방 이동 버튼
- 메시지 삭제: 본인 메시지 길게 누르기 → 삭제 옵션
- 자주 쓰는 멘트: 관리자 채팅 입력 영역 내 단축 문구 버튼 팔레트

**주요 처리**:
- 메시지 소프트 삭제 (삭제된 메시지 처리)
- 자주 쓰는 멘트 목록 저장/조회 (관리자 설정)

---

### 9-5. 회원 개선

**화면/작업 구성**:
- 미승인 학생 입반 전환 화면: 승인 대기 중 학생 목록 → 입반 승인
- 탈퇴 오류 수정: 탈퇴 시 연관 데이터 정리 로직 수정
- 지점 변경 처리: 학생 지점 변경 시 관련 데이터(스케줄·출석) 이관 처리
- 학부모 정렬 기능: 학부모 목록 정렬 옵션 (이름/가입일/지점)

**주요 처리**:
- 입반 전환 승인 로직 및 알림
- 지점 변경 이관 정책 정의 및 구현
