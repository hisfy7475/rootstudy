# 스터디카페 프로젝트 기술스택 회고

## 1. 프로젝트 전체 구조 — 단일 레포

모노레포가 아닌 **단일 Next.js 레포 안에 Expo 앱 폴더를 함께 둔 구조**다.

```
studycafe/
├── src/              ← 웹 앱 (Next.js)
│   ├── app/
│   │   ├── admin/    ← 관리자 화면
│   │   ├── student/  ← 학생 화면
│   │   ├── parent/   ← 학부모 화면
│   │   └── api/      ← 백엔드 API Routes
│   ├── components/
│   ├── hooks/
│   ├── lib/
│   └── types/
└── studycafe-app/    ← 모바일 앱 (Expo) — WebView 쉘
```

모노레포 도구(Turborepo 등)를 쓰지 않고, npm 스크립트(`npm run app`)로 앱 디렉터리를 실행하는 방식으로 연결했다.

---

## 2. 프론트엔드 — Next.js + React Native (Expo)

### 웹 — Next.js

메인 서비스는 전부 **Next.js** 기반 웹이다. 관리자, 학생, 학부모 각각 다른 레이아웃과 라우트를 가진다.

- 관리자: 출결 관리, 회원 관리, 정산, 스케줄, 포인트 등 운영 전반
- 학생/학부모: 출결 확인, 공지, 알림 등 열람 중심

스타일링은 **Tailwind CSS v4**.

### 앱 — React Native (Expo) / WebView 쉘

모바일 앱은 **웹을 WebView로 감싼 구조**다. 네이티브 화면이 별도로 없고, `WebViewScreen.tsx` 하나가 앱 전체를 담당한다.

스토어 배포 목적(알림 푸시 수신 등)이 주된 이유였고, 서비스 로직은 전부 웹에 있다.

---

## 3. 데이터베이스 & 인증 — Supabase

DB, 인증, 파일 스토리지를 모두 **Supabase**로 처리한다.

- **PostgreSQL**: 메인 데이터베이스 (마이그레이션 파일 관리)
- **Auth**: 이메일 로그인 (관리자/학생/학부모 역할 분리)
- **Storage**: 파일 업로드

---

## 4. 백엔드 — Next.js API Routes가 담당

별도의 백엔드 서버 없이 **Next.js API Routes가 백엔드 역할을 한다.**

외부 연동이 여러 개 붙어있어서 API Routes 안에 꽤 많은 서버 로직이 들어가 있다.

| 연동 | 용도 |
| --- | --- |
| **CAPS 출입관리 시스템** | 외부 DB에서 출입 기록 조회 (별도 DB 직접 연결) |
| **NICEPay** | 결제 처리 |
| **네이버 SENS (알림톡)** | 카카오톡 알림톡 발송 |
| **Vercel Cron** | 정기 작업 (자동화) |

특이한 점은 CAPS라는 외부 출입관리 시스템 DB에 직접 연결해서 데이터를 가져오는 구조다. Supabase가 메인 DB고, CAPS DB는 읽기 전용으로 따로 붙어있다.
